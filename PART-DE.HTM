Was ist eine Partition Table?

    Die Partitionstabelle findet sich auf dem ersten Sektor einer jeden
Harddisk. Sie beinhaltet Informationen zur Grösse und Platzierung der
Partitionen auf der Harddisk. Eine Partition ist ein Teil einer Platte;
sie gehört zu einem bestimmten Betriebssystem. Beispiel: das Laufwerk C:
ist in der Partition vom Typ BIGDOS angesiedelt.

    Meist ist die gesamte Festplatte als eine einzige Partition organisiert,
doch will man mehrere Betriebssysteme auf einem Computer installieren, so
muss man seine Festplatte in mehrere Partitionen aufteilen. Daher ist in der
Partitionstabelle auch abgespeichert, welche Partition nun gerade die 'aktive'
ist - diejenige, von der der Rechenr bootet. 

    Wird der Computer angeschaltet, so lädt das BIOS den ersten Sektor
der Harddisk, den sogenannten MBR (Master Boot Record) in der Hauptspeicher.
Neben der Partitionstabelle enthält dieser Sektor einen IPL (Initial Program
Loader), ein Stück Software, das nach der 'aktiven' Partition in der Parti-
tionstabelle sucht und dann den Boot-Sektor des entsprechenden Betriebssystems
in den Hauptspeicher lädt.
    Der normale IPL lädt einfach den Boot-Sektor ohne jede Meldung oder Fragen,
was bei nur einem installierten Betriebssytem durchaus sinnvoll ist. Sind aber
mehrere zum Booten bereit, benötigt man etwas mehr Kommunikationswillen sei-
tens der Maschine. Der mit Partition Manager mitgelieferte IPL kann dem Be-
diener während der Bootverzögerung eine Auswahl an Bootmöglichkeiten präsen-
tieren und sich die Wahl auch gleich merken, sodass das nächste Hochfahren
des Rechners mit demselben Betriebssytem geschieht.
    Die Partitonstabelle kann maximal 4 Einträge aufnehmen. Dieses Programm er-
möglicht es, bis zu 31 bootfähige Partitionen zu erstellen, wenn der
Advanced Boot Manager installiert wird.

Sichern und Zurückschreiben des MBR
    Bevor mit diesem Programm irgendwelche Experimente gemacht werden, sollte
unbedingt eine Sicherung des MBR vorgenommen werden.
    Zuerst sollte eine bootfähige Floppy Disk erstellt werden, auf das dieses
Programm kopiert wird. Danach kann der MBR gesichert werden; entweder geschieht
dies interaktiv im Programm oder auf der Kommandozeile mit
   
           part -s a:\saved.mbr

    Falls irgend etwas schiefgeht, kann jederzeit mit der Floppy gebootet und
der MBR zurückgeschrieben werden:
           part -l a:\saved.mbr
   
    Falls die Diskette auch auf anderen Rechnern benutzt wird, sollte man auf-
passen, nicht versehentlich einen MBR einer anderen Maschine zurückzuschreiben.
Es ist zwar harmlos, aber es könnte zartbeseiteten Gemütern doch ans Lebende
gehen, ein paar Minuten angstvolles Starren auf den roten Bildschirm - und
dann kann man sich in aller Ruhe die Anleitung einverleiben. Trotzdem, daher 
und deswegen ist es ratsam, den gespeicherten MBR's sinnvolle Namen zu geben.
    Ist das obgenannte Szenario dann eingetreten, so erscheint also ein roter
Screen, eine Fehlermeldung vom Programm, dass die Partitionseinstellungen
nicht mit der Geometrie der Festplatte übereinstimmen.




Wahl der aktiven Partition

    Um eine Partition zur aktiven Partition zu machen, kann im interaktiven
Modus die Leertaste gedrückt werden; im Kommandozeilenmodus sieht das so aus:
   
            part -p        Partitionstabelle anzeigen (print)
            part -a n      aktiviere Partition n
      
Falls ein IPL installiert wurde, so kann die aktive Partition mit der Leertaste
deaktiviert werden - wird nun neu gebootet, so fährt der Rechner von der zwei-
ten Platte oder vom Floppy-Laufwerk hoch.




Advanced Boot Manager

    Mit dem Advanced Boot Manager ist es möglich, bis zu 31 bootfähige
Partitionen zu erstellen. Er hält seine eigene Partitionstabelle in einem
separaten Teil der Festplatte.
    Wenn der Rechner gebootet wird, versucht der erste Teil des Advanced Boot
Managers (er sitzt im MBR), den zweiten Teil von der Festplatte zu lesen und
prüft dessen Integrität. Ist dieser Teil defekt, so fordert er den Benutzer
auf, von Diskette oder Festplatte zu booten.
    Danach sucht der Advanced Boot Manager nach Viren und vergleicht dann
die Einträge im MBR mit seiner eigenen Partitionstabelle. Falls irgendwelche
Änderungen vorgenommen wurden, kopiert der Advanced Boot Manager diese in seine
eigene Partitionstabelle und fordert den Benutzer auf, part.exe auszuführen, um
die Änderungen angezeigt zu bekommen. Siehe hierzu auch die Bemerkungen zur
Installation eines neuen Betriebssystems.
    Der Advanced Boot Manager präsentiert nun ein Menu, in dem die Partition
markiert werden kann, von der gebootet werden soll. Nachdem die Auswahl erfolgt
ist, wird der Eintrag der selektierten Partition in die MBR Tabelle kopiert; der
Rest der Harddisk wird versteckt.
    Die Eingaben für den Advanced Boot Manager sind:  

    ENTER - Booten von der markierten Partition (Tasten 1-9 funktionieren auch)
    ESC   - Booten von der markierten Partition, ohne dass sich der Advanced
            Boot Manager für das nächste Booten merkt
    A     - Booten von A und die markierte Partition in den MBR schreiben
    H     - Booten von A, aber alle Partitionen der Harddisk verstecken,
            mit Ausnahme des ersten Teils des unbenutzten Platzes, der grösser
            als 63 Sektoren ist. (um ein neues OS zu installieren).
    TAB   - Booten von der nächsten Festplatte.

    ALT-ENTER - Wie ENTER, aber ohne die im Boot-Manu angegebenen Tastatur-
            eingaben in den Keyboard-Buffer zu legen.

Advanced Boot Manager Menu bearbeiten

  Für jeden Menu-Eintrag kann man folgendes spezifizieren:   (Beispiel)
  
  - Gerät, von dem gebootet werden soll (Dev)

       C - aktuelle Harddisk (worauf dieses Menu residiert)
       D - nächste Harddisk
       A - erstes Floppy Laufwerk
  
  - Partitionen, die aktiviert werden sollen (0 - keine)
  
  - Reihe, in die die Partition plaziert werden soll.        (Beispiel)

  - Welche Partitionen zusammen mit der Bootpartition gezeigt werden sollen.  
    Zur Verfügung stehen: Last, Previous, Next, und Last3,
    einschliesslich der Bootpartition.

  - Passwortschutz für die Partition (PP)

  - Der gewünschte Name der Partition im Auswahlmenu
  
  - Tastatureingaben, die in den Keyboardbuffer gelegt werden sollen, bevor
    das Booten beginnt.
    (Beispiel: F4 vor dem Booten mit Win95 veranlasst ein Booten mit der vor-
    gängigen MS-DOS Version)

    
  Die Boot Manager Optionen können mit Alt-O geändert werden:
 
       - Boot Menu Titel
       - Boot Virus Check aktivieren
       - Vor dem Anzeigen des Boot Menus den Bildschirm löschen
       - Timeout (0 beendet das Herunterzählen und wartet auf eine
         Benutzereingabe)
       - Standard Partition (überschreibt die gemerkte "zuletzt gewählt")
       - Optionales Passwort für den Advanced Boot Manager 
       - Ignorieren der ungenutzten Partitionen, wenn "Prev", "Next", "Last"
         evaluiert wird
    Ein gesetztes Passwort schützt vor dem Booten vom Floppy (via Taste A, H)
und von der zweiten Festplatte via TAB, Es ist jedoch möglich, einen Menu-
eintrag einzugeben, um von Floppy oder von der zweiten Festplatte zu booten,
ohne Passwortschutz: das PP - password protected - flag einfach nicht setzen.




Setup-Beispiel Advanced Boot Manager
Angenommen, es befinden sich zwei Harddisks im System:

    Die neue 4.3Gb Harddisk                 Eine alte 340Mb Harddisk

    1  AdvBootManager    30k                1  DOS FAT-16     340Mb
    2  Windows FAT-32  2600M
    3  Windows NT NTFS  400M
    4  Linux            400M
    5  Linux swap        30M
    6  DOS Extended     800M
 
 Damit könnte das Advanced Boot Manager Menu etwa so aussehen:
 
    #  Dev  Partitions  Name                    Keys
     
    1   C   2/*  Last   Windows 95                N
    2   C   2/*  Last   Windows 95 / Prompt       Y  <-  "F8 6 Enter"
    3   C   3/2  Last   Windows NT                N
    4   C   4/3  Next   Linux                     N
    5   C   4/3  Last3  Linux (sees extended)     N
    6   D   6/*  ----   Old MS-DOS (sees ext)     N
    7   D   0    ----   Old MS-DOS (sees nothing) N
    Bemerkung: Damit DOS auch von der zweiten Festplatte gestartet werden kann,
sind folgende Schritte nötig:
       
   - Der Compact oder Advanced Boot Manager ist auf der zweiten Festplatte zu
     installieren
   - Im Setup Screen für die FAT-16 Partition ist die Drive Number auf 129 zu
     setzen.
   - Kontrollieren Sie, dass Sie keine nicht-versteckten primären FAT-16
     Partitionen auf der ersten Festplatte haben, wenn Sie von der zweiten booten.

    Zu beachten ist ferner, dass Windows NT und Linux in den Reihen 2 und 3 der
Partitionstabelle plaziert werden, wobei es keine Rolle spielt, wohin Win95 hinkommt.
    Für Windows NT ist es wichtig, immer in derselben Reihe zu sitzen; seine boot.ini
hat nämlich eine die folgende Syntax:
 
multi(0)disk(0)rdisk(0)partition(2)\WINNT="Windows NT" /basevideo /sos

und es wird seinen Kernel bestimmt nicht finden, wenn sein Platz in der Partitions-
tabelle verschoben wird.



Anderes Beispiel - Vorbereitung für die Installation von Win98 (oder anderes OS)
   
   - Erstellen Sie eine Bootdiskette, auf der Partition Manager drauf ist.
   - Erstellen Sie die Partition und formatieren Sie sie.
   - Editieren Sie das Bootmenu

           Ihre Partitionen

      1  AdvBootManager    30k
      2  Windows FAT-32  1600M     <-- Windows 95 Partition
      3  Windows FAT-32  1200M     <-- die neu erstellte Partition
      4  DOS Extended     800M       \ formatiert für Windows 98
 
      Advanced Boot Manager Menu
 
      #  Dev  Partitions  Name                    Keys
      
      1   C   2/*  Last   Windows 95                N
      2   C   2/*  Last   Windows 95 / Prompt       Y  <-  "F8 6 Enter"
      3   C   3/*  Last   Windows 98                N

   - Speichern Sie diese Daten auf Festplatte und auf Floppy
   - Maschine neu booten
   - Im Bootmanager Menu "Windows 98" wählen und A drücken
     um von Floppy a: das "Windows 98 Setup" zu starten 
   - Nach der Installation von Windows 98 starten Sie den Partition Manager
     um etwelche Änderungen im MBR durchzusehen und reinstalliern den Boot-
     manager falls nötig.




Compact Boot Manager

   Der Compact Boot Manager passt in 446 Bytes hinein und braucht keinen
Platz auf der HD über dem für einen normalen MBR vorgesehenen Platz.
Er ist einfach ein IPL.

Beim Installieren haben Sie 4 Optionen: 

    - Einschalten des BootViruses Checks
    - Time-Out für die Boot Auswahl
    - Standard Partition (überschreibt den Eintrag "last selected")
    - Standard Boot Partition, falls keine der Partitionen aktiv ist.
      Die Möglichkeiten sind: A: und "next hard drive".

Tip:  Falls die Standard Partition auf A gesetzt wird, wird der Rechner 
      immer von A: booten, falls Tab gedrückt wird - von der nächsten HD,
      und wenn Space gedrückt wird - dann wird der Time-Out angehalten und
      Sie können wählen.




Meldungen des Compact Boot Manager

       Meldung                         Beschreibung

                          ESC um das OS zu booten
                          SPACE um Time-Out zu stoppen und Auswahl zu treffen
"Booting from: HD1/1..."  1-4 drücken um von der 1-4 Partitionzu booten
"Booting from: HD1/_"     A um vom Floppy zu booten oder TAB um von der  
                          nächsten Harddisk. Jede andere Taste wird das
                          Booten fortfahren lassen und bleibt im Puffer.
                          zb. F4, F5, F8, ... (siehe Tips)

"Writing changes..."      Der IPL speichert Ihre Auswahl im MBR.

"Error!"                  Fehler beim Lesen der Disk oder leere Partition.

"Virus !!!"               Es wurde ein IPL gefunden, bei dem ein Interruptvektor
                          nocht ins ROM BIOS Memory zeigt (siehe Viren-Schutz).
                          Sie können ENTER drücken um zu booten, aber es wäre
                          vernünftiger, den Rechner auszuschalten und ein
                          Antivirus-Programm laufen zu lassen, nachdem von
                          einer sauberen Floppy gestartet wurde.




Viren-Schutz

   Der IPL vom Partition Manager kann Viren erkennen noch bevor das 
ein Betriebssystem ins Memory geladen wird. Dies ist wichtig, weil
MBR Viren von Antivirenprogrammen nicht erkannt werden können, wenn sie
nach dem Betriebssystem geladen werden.

   Die Virenerkennung ist implementiert, indem 29 Interrupt Vektoren
von Int 0h bis Int 1Ch getestet werden. Diese müssen auf C000:0000h oder 
höher zeigen, wo also das ROM BIOS sitzt. Diese Interrupts kontrollieren
alle lebenswichtigen Funktionen des Systems, so etwa Timer, Video, Keyboard
und Disk.
   
   Falls einer dieser Interrupts geändert wurde, so ist es sehr wahrscheinlich,
dass ein Virus das System infiziert hat. In diesem Falle hält der IPL und
gibt eine Warnung aus. Man kann nun diese Warnung ignorieren und mit dem Virus
booten oder neu booten und ein Antivirus-Programm laufen lassen. Siehe hierzu
IPL Meldungen.

   Auf manchen Sytemen kann es aber auch zu Fehlalarm kommen. In diesem Falle
können Sie die Virenwarnung deaktivieren.

   Eine andere Möglichkeit, sich vor Viren zu schützen, ist das Verhindern von
ungewolltem Booten von einer Floppy. Mit diesem Programm ist dies möglich: und
dennoch kann von einer Floppy gebootet werden, wenn man dies wünscht. Um dies
zu erreichen, ist folgendes zu tun:

   1. Installieren Sie entweder den Advanced oder Compact Boot Manager.

   2. Die Boot-Reihenfolge ist auf C:,A: im BIOS Setup zu ändern. 
    
   3. Falls man von A: booten möchte, drückt man zur Bootzeit A.

   Dieser Trick bewahrt Sie nicht nur vor Viren, sondern erhöht auch die
Arbeitsgeschwindigkeit. Eindrücklich zeigt sich dies vor allem bei der Kombina-
tion von QEMM 8.xx zusammen mit MS-DOS oder Windows 95. 

   Setzen Sie dafür die Option "BootFloppy:N" in QEMM's Quick Boot und fügen
sie "SWITCHES=/F" in die CONFIG.SYS ein.

   Beim nächsten Reboot wird QEMM's Quick Boot den BIOS Memory Test über-
springen und führt Sie direkt zum Compact Boot Manager Prompt, wo Ihnen ein
komfortables Booten mittels F4, F5, oder F8 zur Verfügung steht ...




Neue Partition einrichten

   Um eine neue Partition einzurichten suchen Sie einen unbenutzten Eintrag
in der Partitionstabelle und drücken INS, dann wählen Sie den gewünschten
Typ des Filesystems.
   
   Danach wird der Partition Manager automatisch die richtigen Werte für
Zylinder, Heads und Sektoren eintragen. 
   
   Normalerweise müssen Sie diese Werte nicht ändern. Falls Sie aber eine
eigene Vorstellung des Partitionierungsschemas habe, so können Sie diese
Werte ändern. Machen Sie sowas, so ist es empfohlen, jedoch nicht notwendig,
Start und Ende an den Grenzen vom Zylinder (oder Head) zu setzen.
Beispiel:
Die Harddisk hat 16 Heads und 63 Sektoren pro Head, dann sollte die Partition
bei Head 0, Sektor 1 beginnen und bei Head 15, Sektor 63 aufhören.

   Falls Sie mehrere Betriebssysteme wie MS-DOS, Windows 95 oder Windows NT
installieren möchten, dann sollten Sie aufpassen, dass sie sich nicht in die
Quere kommen: Verstecken Sie die vorhandenen Partitionen mit Ausnahme der zu
installierenden, installieren das neue OS und holen die versteckten Partitionen
wieder zurück (unhide). 

   Solange Sie Zahlen eingeben, können sich die Partitionseinträge rot färben:
machen Sie sich darüber keine Gedanken bevor sie das Partitionieren nicht
abgeschlossen haben.

   Rote Partitionseinträge bedeuten, dass die Einstellungen ungültig sind oder
in Konflikt zu der momentanen Disk-Geometrie stehen.

   Es gibt vier Error-Codes, um herauszufinden, was falsch ist. Sie werden rot
angezeigt, wenn der Cursor über den ungültigen Partitionseintrag geführt wird.
      
mbr: die Partition überlappt den MBR im ersten Sektor der Disk oder
     eine Extended DOS Partition.
        
range: die Partition hat Werte für Zylinder, Head, oder Sektor (CHS)
     ausserhalb des gültigen Bereichs der Disk oder der Extended DOS Partitions-
     grenzen oder der End-Werte ist kleiner als der Begin-Wert. Bewegen Sie den
     Cursor auf die CHS-Felder, um die erlaubten Werte zu sehen.

overlapped: zwei Partitionen überlappen sich.
   
inconsistent: diese Meldung bedeutet, dass die CHS-Werte nicht mit dem Start-
     Sektor und der Partitionsgrösse übereinstimmen, was auch in der Partitions-
     tabelle abgespeichert ist. Dieser Fehler tritt normalerweise auf, wenn
     die Partitionstabelle beschädigt ist oder wenn das BIOS die Disk-Geometrie-
     Übersetzung seit der letzten Partitionstabellenmodifizierung geändert hat.
     Es kann auch sein, dass der Wert 0 für Sektor eingegeben wurde (Sektor muss
     mindestens 1 sein).
        
   Falls die rote Farbe schon beim Starten auftritt, so ist es wahrscheinlich,
dass der Disk-Controller gewechselt wurde, die BIOS-Einstellungen geändert wurden
oder der MBR beschädigt ist. Falls dem so ist, hantieren Sie nicht an den Werten
herum, ausser Sie sind sich absolut sicher, was Sie tun.
   Ratschlag für Profis: Es spielt keine Rolle, wie das BIOS die Disk-Geometrie 
anzeigt, die relative Zahl von Sektoren und die Partitionsgrösse bleiben immer
gleich. Um das Inkonsistenzproblem zu beheben, muss man den LBA Mode per Taste F4
einschalten, die Partitionsgrösse notieren, ändern und sie dann erneut auf den
notierten Wert (die korrekte Grösse) ändern. Das wird das Programm zwingen, die
Werte im CHS neu zu berechnen und zwar in Übereinstimmung mit der momentanen
Disk-Geometrie.
   Beachten Sie zwei Dinge: Soweit ich das beurteilen kann, benutzt DOS die 
CHS Werte zum Booten und nicht die "Start Sektor/Anzahl-Sektoren" Einträge.
Es benutzt die Felder "sectors prior partition" und "big total number of sectors"
aus seinem Boot-Eintrag (Record).

   Falls Sie OnTrack DDO oder Ez-Drive benutzt haben, dann wissen Sie, dass diese
die Disk "verdrehen": OnTrack um 63 Sektoren und Ez-Drive um 1 Sektor. Daher müssen
sie diesen Umstand im Boot-Record für DOS mitberücksichtigen.




Formatieren einer Partition
      
   Partitionen können formatiert werden oder nur auf fehlerhafte
Sektoren geprüft werden - ohne die Daten zu zerstören. 
   
   Um eine Partition zu verifizieren, drücken Sie V im interaktiven
Modus oder führen part wie folgt aus (n = Nummer der Partition):
      
          part -v n

   Um eine Partition zu formatieren, drücken Sie F im interaktiven
Modus oder führen part wie folgt aus (n = Nummer der Partition):
     
          part -f n options ...
      
   Die Format-Optionen hängen vom gewünschten Filesystem ab.
   
Wichtige Bemerkung zum Formatieren von DOS Partitionen

   Erstens: DOS sieht keine neu erstellten Partitionen, bevor der Rechner
nicht neu gestartet wurde, weil es die Partitionstabelle nur einmal scannt.
   Wurde also eine Partition erstellt und/oder formatiert, so muss der
Rechner neu gebootet werden.
   
   Falls die Partition schon montiert wurde (man kann sie als DOS-Laufwerk
sehen) und sie wurde mit anderen Parametern neu formatiert, so ist ebenfalls
ein Neustart notwendig, weil DOS die alten Werte im Memory behält und die
Harddisk mit Bestimmtheit in Unordnung bringen wird, wenn man auf diese
schreibt, ohne den Rechner zu booten.

   Das zweite Problem ist der Disk-Cache. Partition Manager leert diesen mit
Hilfe des SmartDrive v 4.00+ API. Die meisten neuen Cache-Programme, wie
etwa PC-Cache 8.0, NCACHE2 oder SpeedDrive unterstützen dies. Falls aber
andere Cache-Manager zum Einsatz kommen, sollten sie vor der Formatierung von
DOS-Partitionen abgeschaltet werden oder - booten Sie den Rechner nach einer
Formatierung neu. 

   Der Unterschied im Verhalten von part.exe und format.com ist, dass part
Partitionen anhand der Nummer in der Partitionstabelle formatiert, format
hingegen anhand des Buchstabens in der DOS Device Liste. Daher sollten vor
einem Neustart keine Programme benutzt werden, die Partitionen via DOS
ansprechen; Programme, die Partitionen ansprechen, ohne DOS zu bemühen, können
problemlos benutzt werden; zB. zpart, das ganze Partitionen (de)komprimiert
(von) in Files.

   Einschränkung: Momentan kann part logische DOS Drives nur im interaktiven
Modus formatieren.




Wichtige Anweisung zur Installation eines neuen OS

   Windows 95 und einige andere Betriebssysteme ersetzten bei der
Installation den IPL mit ihrem eigenen Code. Ist der Compact oder Advanced
Boot Manager auf der Festplatte installiert, so wird dieser beim Booten nach
einer Installation eines solchen OS nicht mehr erscheinen.

   Ich habe mich um diese Sache gekümmert. Es sollte nicht zu Datenverlust
führen, auf alle Fälle nicht mit Windows 95. Im schlimmsten Falle müsste
part.exe ausgeführt werden und der Boot Manager neu installiert werden. Um
sicher zu gehen, dass nichts verloren geht, halten Sie sich an diese Vorgehens-
weise, um ein neues OS zu installieren:
   
   - Erstellen Sie eine bootfähiges Floppy und kopieren Sie den Partition
     Manager darauf.
   - Starten Sie den Partition Manager und erstellen Sie die Partition für das
     neue OS.
   - Falls möglich, formatieren Sie diese Partition mit dem Partition Manager.
   - Im Advanced Boot Manager editieren Sie das Boot Menu (siehe Beispiel)
   - Speichern Sie die Partition Manager Daten auf die eben erstellte Floppy.
   - Starten Sie den Rechner neu und installieren Sie das neue OS.
   
   Falls nun der MBR vom neuen OS modifiziert wurde, schauen Sie sich die
Änderungen an und installieren den Boot Manager erneut. (Drücken Sie F2 um den
MBR zu sichern und den ausführbaren Advanced Boot Manager Code auf Disk zu
installieren).

Nicht-benutzte Partition

   Dies zeigt einfach an, dass der Eintrag in der Partitionstabelle
verfügbar ist. Es sagt nichts darüber aus, wieviel Platz auf der Disk
vorhanden ist - das muss man selber herausfinden.

   Wenn man die Start- und End-Werte der Partition angibt mit Typ "Unused",
so wird der Partition Manager ein Formatieren und Verifizieren des Diskspace
erlauben.
   
   Format kenn zwei Optionen: "/destructive" oder "/clean". Die erste führt
eine Low Level Disk Formatierung durch, die zweite überschreibt den Disk
Space mit lauter Nullen.



Klassische MS-DOS Filesysteme

     0x01, "DOS FAT-12"
     0x04, "DOS FAT-16 (unter 32M)"
     0x06, "BIGDOS FAT-16 (über 32M)"

Alle Filesysteme haben dieselbe Struktur:
      
     [Boot Sektor]
     [Erste Kopie der FAT]
     [Zweite Kopie der FAT]
     [Wurzelverzeichnis]
     [Cluster 2]
     [Cluster 3]
     [Cluster 4]
     ...

   Der Boot Sektor, die FATs, das Wurzelverzeichnis und die Cluster haben
eine feste Grösse: sie können nur mit anschliessendem Formatieren ge-
ändert werden.
   Platz für alle Unterverzeichnisse und Dateien wird von den Clustern
alloziert. Ist die Clustergrösse 8 Sektoren = 4,096 Bytes, dann benötigt
auch eine Datei mit nur einem Byte einen solchen Cluster von 4,096 Bytes.
   Für jeden Cluster existiert ein Eintrag in der FAT (File Allocation
Table), der darüber Auskunft gibt, ob der Cluster frei, alloziert oder
beschädigt ist. Daher bestimmt die Grösse der FAT die maximale Anzahl
Clusters.

   Das FAT-12 Filesystem ist auf 4084 Clusters beschränkt. Üblicherweise
wird es für Disketten gebraucht, aber auch kleine Harddiskpartitionen sind 
denkbar.
   
   FAT-16 wurde auf den ersten Festplatten benutzt. Die originale Version
konnte bis zu 65,000 Cluster auf Partitionen bis zu 32M aufnehmen.
   
   BIGDOS FAT-16, welches mit DOS 3.31 freigegeben wurde, hat die 32M 
Beschränkung nicht mehr: es kann bis 2T aufnehmen. Für die Rückwärts-
kompatibilität akzeptieren alle DOS Versionen ab 3.31 FAT-12 und FAT-16
Filesysteme innerhalb der BIGDOS Partition.

   In diesem Programm ist ein Formatierungsutility eingebaut, das alle drei
Filesytem-Typen formatieren kann mit den folgenden Optionen:

   part -f n [/quick | /destructive] [/l:volume_label]
             [/c:sect_per_cluster] [/x:max_num_sect | /x:disk]

   Wird die Option "quick" gewählt, so wird angenommen, dass keine
fehlerhaften Sektoren auf dem Laufwerk sind. Man kann die Disk einmal veri-
fizieren und - falls es keine fehlerhaften Sektoren gibt - diese Option
verwenden.
 
   Wird keine Clustergrösse angegeben, so wird diese automatisch in
Abhängigkeit von der Partitionsgrösse von 4 Sektoren und darüber selektiert.
Eine Tabelle für das BIGDOS FAT-16 Filesystem:

             Cluster-Grösse         Maximale Grösse der Partition

        1 Sektor        512 Bytes           32M
    *   2 Sektoren    1,024 Bytes           64M         empfohlene
    *   4 Sektoren    2,048 Bytes          128M       Cluster-Grössen
    *   8 Sektoren    4,096 Bytes          256M        sind mit '*'
    *  16 Sektoren    8,192 Bytes          512M          markiert
    *  32 Sektoren   16,384 Bytes            1G
    *  64 Sektoren   32,768 Bytes            2G
      128 Sektoren   65,536 Bytes            4G

Obwohl man eine Disk mit jeder dieser Cluster-Grössen formatieren kann,
ist es nicht ratsam, sehr grosse oder sehr kleine Werte zu verwenden.
Viele Programme zeigen damit ein eigenartiges Fehlverhalten. Beispiels-
weise blockiert SCANDISK mit einer Clustergrösse von 128, Norton Disk
Doctor scheint ohne Probleme zu laufen.

   Mit dem max_num_sect Parameter kann Platz in der FAT für zukünftige
Erweiterungen der Partition reserviert werden. Wird die Option "/x:disk"
(genau so wie geschrieben) verwendet, wird Format die FAT genügend gross
anlegen, so dass die Partition auf die gesamte Disk vergrössert werden
kann.
   
   Man kann auch FAT-Partition verkleinern (schrumpfen). Um dies zu
bewerkstelligen, starten Sie einen Defragmentierer (etwa DEFRAG.EXE). Dann
gehen Sie zum Setup Screen in part.exe. Eine der Zeilen zeigt die minimale
Partitionsgrösse, basierend auf dem letzten mit Daten belegten Cluster. Der
Wert sollte gleich der Summe der Datei- und Verzeichnisgrössen auf der Hard-
disk sein. Ist er grösser, so bedeutet das, dass einige versteckte (hidden)
Dateien vom Defragmentierer nicht verschoben wurden. Nun müssen diese Dateien
gesucht werden (zB. Norton Speed Disk zeigt ihre Namen), die System- und
Hidden-Attribute entfernt werden und der Defragmentierer erneut gestartet
werden. Vorsicht beim verschieben von solchen Dateien: einige Programme
benutzen die absolute Position eines Files, um illegalen Installationen
vorzubeugen.



Windows 95 FAT-32 Filesystem

     0x0B, "Windows 95 FAT-32"

   Diese Filesystem wurde geschaffen, um Disks mit einer Kapazität von
mehr als 2G zu unterstützen, welches die Grenze des FAT-16 Filesystems ist.
FAT-32 Partition können 2T (2,048G) gross sein. 

Es hat die folgende Struktur:
      
     [Boot Record (3 Sektoren)]
     [Backup Kopie des Boot Records]
     [Erste Kopie der FAT]
     [Zweite Kopie der FAT]
     [Cluster 2]
     [Cluster 3]
     [Cluster 4]
     ...

   Das Wurzelverzeichnis (Root directory) auf FAT-32 wird wie ein normales
Verzeichnis behandelt, kann überall auf der Disk liegen und hat keine vor-
definierte feste Grösse (im Gegensatz zu FAT-16).

   Die Optionen für die Formatierung mit dem FAT-32 Filesystem sind
(ähnlich FAT-16):

   part -f n [/quick | /destructive] [/l:volume_label]
             [/c:sect_per_cluster] [/x:max_num_sect | /x:disk]

   Die Strategie zur Grössenänderung einer FAT-32 Partition ist ebenfalls
ähnlich wie bei FAT-16:
   
   - Lassen Sie einen Defragmentierer auf die FAT-32 Partition los.
   - Ändern sie die Partitionsgrösse in der Partitionstabelle.
   - Wechseln Sie zum Setup Screen und nehmen sie die selben Änderungen hier
     vor (prüfen Sie sie auf Gültigkeit - sie werden sonst rot dargestellt)
   - Booten Sie den Rechner neu und führen Sie ScanDisk aus, um die Disk zu
     checken und um den Betrag des freien Speicherplatzes zu resetten.




Extended (erweiterte) DOS Partition

   Eine Partitionstabelle bietet Platz fuür vier Eintrage. Um mehr als vier
Partitionen zu benutzen, bedient sich DOS des Extended DOS Partitionen Schemas.
Dieses erlaubt es bis zu 23 logische DOS Laufwerke auf einem physikalischen
Laufwerk zu haben. Leider schränkt das native DOS Programm FDISK die Anzahl
der primären (also bootfähigen) Partitionen auf gerade mal eins pro Harddisk ein.
In Wirklichkeit KANN man mehr als eine primäre Partiton haben. Das Problem ist,
dass das MS-DOS FDISK sich beharrlich weigert, eine zweite solche Partition an-
zulegen.
   
   FDISK erstellt nur eine primäre Partition und markiert den verbelibenden Rest
des Platzes als ein Extended Partition. Der erste Sektor der Extended Partition
wird EMBR (Extended MBR) genannt und dieser hat dieselbe Struktur wie der MBR.
Innerhalb jeder Extended Partition kann eine FAT (oder NTFS) Partition sein
(genannt Logisches Laufwerk) und eine weitere Extended Partition. Aus unersich-
lichen Gründen werden die relativen Sektoren der inneren Extended Partitionen vom
Start der äussersten Partition berechnet, die relativen Sektoren der FAT Partition
jedoch vom Start der einschliessenden Extended Partition.

Hier ein Beispiel dazu:

   HD  520 Mb
   ---------------------------------------------------
   Primary FAT-16  120Mb                                [ C: 120Mb ]
   Extended DOS    300Mb -----------------------------
                         FAT-16  100Mb                  [ D: 100Mb ]
                         ExtDOS  200Mb ---------------
                                       FAT-16  200Mb    [ E: 200Mb ]
                                       ---------------
                         -----------------------------
   Linux           100Mb                                [ /  100Mb ]
   ---------------------------------------------------

   Zu bemerken ist, dass die Reihenfolge der Partitionen auf der Disk ebenfalls
eine Rolle spielen. FAT-Partition müssen am Anfang stehen. Und die Extended
Partition muss folgen.
   
   Tip: Im Partition Manager kann man PgUp und PgDn benutzen um zwischen den EMBR
Levels zu wechseln.

   Man kann weder von Extended Partitionen booten, noch von etwelchen Logischen
Laufwerken innerhalb dieser. Falls man es trotzdem versucht, so hängt sich das 
System einfach auf, weil dieser Boot Record im Gegensatz zu anderen Filesystemen
keinen Loader Code enthält. An dessen Paltz stehen nur Nullen. Man könnte einen
"nicht-bootbaren" Dummy Boot Record installieren, der anstatt zu crashen eine
Meldung ausgibt:
  
      Extended DOS partition is not bootable.
      Press any key to reboot...
   
Bootfähigkeit von Logischen Laufwerken

   Wie weiter oben erwähnt, ist es nicht möglich von einer Extended Partition
oder von einem Logischen Laufwerk darin zu booten. Was aber, wenn man muss? Die
Lösung ist einfach. Alles, was zu tun ist, ist den Record, der das Logische
Laufwerk beschreibt, vom EMBR in den primären MBR zu schreiben und die Grösse
der Extended Partition anzupassen. Dann muss man die korrekte Anzahl der 
"Sektoren vor der Partition" im DOS Boot Sektor setzen, dessen Wert gebraucht
wird, um die System Files zu laden. Schliesslich ist sys.com auszuführen, um
die System Files auf Disk zu schreiben.

Sehen wir mal, was wir mit unserem Beispiel tun können.

Variante 1.
  
   Zu bemerken ist, dass die Grösse der Extended Partition geändert wurde
und die zweite Extended Partition verschwunden ist.

   HD  520 Mb
   ---------------------------------------------------
   Primary FAT-16  120Mb                                [ C: 120Mb ]
   Extended DOS    100Mb -----------------------------
                         FAT-16  100Mb                  [ D: 100Mb ]
                         -----------------------------
   Primary FAT-16  200Mb                                [ E: 200Mb ]
   Linux           100Mb                                [ /  100Mb ]
   ---------------------------------------------------
      

Variante 2.

   Beide Eintrage (Records) wurden von der Extended Partition in den MBR
verschoben.

   HD  520 Mb
   ---------------------------------------------------
   Primary FAT-16  120Mb                                [ C: 120Mb ]
   Primary FAT-16  100Mb                                [ D: 100Mb ]
   Extended DOS    200Mb -----------------------------
                         FAT-16  200Mb                  [ E: 200Mb ]
                         -----------------------------
   Linux           100Mb                                [ /  100Mb ]
   ---------------------------------------------------
      

Variante 3.
  
   HD  520 Mb
   ---------------------------------------------------
   Primary FAT-16  120Mb                                [ C: 120Mb ]
   Primary FAT-16  100Mb                                [ D: 100Mb ]
   Primary FAT-16  200Mb                                [ E: 200Mb ]
   Linux           100Mb                                [ /  100Mb ]
   ---------------------------------------------------
 




Kommandozeilen-Optionen

Usage: part [-q] [-d disk] [-command ...]

 part          - Interaktiver Modus
 part -i       - IDE Disk Information drucken
 part -p       - Partitionstabelle drucken
 part -p -r    - Info rekursiv ausgeben
 part -a n     - Aktivieren der n-ten Partition
 part -v n     - Verifizieren der n-ten Partition
 part -f n ... - Formatieren der n-ten Partition
 part -s file  - MBR in Datei speichern
 part -l file  - MBR aus Datei laden
 part -c file  - MBR mit Datei vergleichen
 part -Reboot  - Rechner rebooten

 Das Programm gibt einen Exit Code 0 zurück, falls kein Fehler aufgetreten
 ist und/oder die verglichenen Objekte identisch waren (sonst wird 1
 zurückgegeben).
